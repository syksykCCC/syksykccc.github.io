# 期中考试模拟试题 5：选择题（Lec1-Lec13）

- 题型包含单选题与不定项选择题；不定项题可能有一个或多个正确选项。

## 一、选择题

### 选择题 1（不定项）：地址空间分段内容

关于进程地址空间中的常见区域，以下哪些说法正确？

A. Code/text 段保存机器指令，通常也可包含只读常量。  
B. Static data 段保存全局变量和静态变量。  
C. Stack 通常保存调用帧、返回地址、参数和局部变量。  
D. Heap 通常用于动态分配对象，如 `malloc` 返回的内存。  
E. 每个线程共享同一个栈，因此局部变量天然共享。

:::tip 答案与解析
正确选项：A、B、C、D。

同一进程的线程共享地址空间中的 code/data/heap，但每个线程通常有独立栈和寄存器上下文。局部变量通常位于各自线程栈上，不是“天然共享”。
:::

### 选择题 2（单选）：地址访问的可能语义

对某个虚拟地址执行读写，以下哪一项说法最准确？

A. 一定对应普通 DRAM 读写。  
B. 一定触发系统调用。  
C. 可能是普通内存、memory-mapped I/O、无效访问 fault，或因权限不足 trap。  
D. 用户程序访问任意地址都由编译器静态阻止。

:::tip 答案与解析
正确选项：C。

地址访问的语义取决于映射和权限。合法页可能是普通内存，也可能映射设备；无效或越权访问会触发异常交给 OS。
:::

### 选择题 3（不定项）：模式切换类型

课程区分 syscall、interrupt、trap/exception。以下哪些匹配正确？

A. `read(fd, ...)` 进入内核服务通常属于 syscall 路径。  
B. 定时器到期抢占当前线程属于 interrupt。  
C. 除零或非法地址访问属于 trap/exception。  
D. syscall、interrupt、exception 都是受控进入内核的控制转移。  
E. interrupt 必须由当前运行程序显式执行指令触发。

:::tip 答案与解析
正确选项：A、B、C、D。

interrupt 是外部异步事件，如定时器和设备；不要求当前程序显式执行触发指令。syscall 是主动请求服务，trap/exception 是当前指令引发的同步异常。
:::

### 选择题 4（单选）：interrupt vector 的作用

硬件在发生 interrupt/exception/syscall 入口时，使用事件编号索引 interrupt vector 的主要目的是：

A. 找到对应内核处理函数入口。  
B. 在用户栈上创建普通 C 函数调用。  
C. 自动修复用户程序 bug。  
D. 计算文件 offset。

:::tip 答案与解析
正确选项：A。

interrupt vector 是按事件编号索引的特权表，记录内核处理入口及相关属性。
:::

### 选择题 8（单选）：一次 `read` 的返回值

下面代码：

```c

ssize_t n = read(fd, buf, sizeof(buf));
```

若 `fd` 为普通或类文件描述符，最准确说法是：

A. 成功时 `n` 一定等于 1000。  
B. 成功时 `n` 可能小于 1000。  
C. `read` 不会返回 0。  
D. 失败时 `n` 一定等于 0。

:::tip 答案与解析

### 选择题 13（单选）：ThreadRoot 的作用

### 选择题 16（不定项）：信号量与锁如何选择

以下哪些说法符合课程中的锁/信号量区分？

A. 锁主要用于互斥保护临界区。  
B. 信号量可用于表达资源计数和调度约束。  
C. 二值信号量在某些场景可类似锁使用，但语义上信号量更一般。  
D. 任何信号量方案都比管程更易验证。

### 选择题 17（单选）：Hoare 管程语义

Hoare 管程语义下，线程 `signal` 一个条件变量时，典型语义是：

A. 发 signal 线程继续持锁执行，等待线程只是进入 ready queue。  
B. 立即把锁和 CPU 交给被唤醒线程，使其在条件仍成立时运行。  
C. signal 永远被记录为计数，未来 wait 一定立即返回。

:::tip 答案与解析

### 选择题 18（不定项）：语言级同步支持

关于不同语言/库中的同步支持，以下哪些说法正确？

A. C 中手工加锁/解锁容易在错误路径漏解锁。  
B. C++ `std::lock_guard` 利用 RAII，可在作用域退出时自动释放锁。  
C. Python 的 `with lock:` 也体现作用域化释放思想。  
D. Java `synchronized` 可提供内建管程风格支持。

:::tip 答案与解析
正确选项：A、B、C、D。

RAII/作用域化释放能减少漏解锁，但不能自动解决锁顺序错误、死锁、饥饿和条件变量谓词错误。
:::

### 选择题 19（单选）：test-and-test-and-set 的好处

相比每轮都执行 `test_and_set` 的自旋锁，test-and-test-and-set 的主要好处是：

A. 先用普通读自旋等待锁看起来空闲，减少反复写锁变量造成的缓存一致性流量。  
B. 完全不需要原子指令。  
C. 只能在单核上使用。  
D. 会自动阻塞线程而非忙等。

:::tip 答案与解析
正确选项：A。

`test_and_set` 是写性 RMW，会造成缓存行迁移。先 read-spin 可减少高竞争下的一致性流量，但它仍是自旋锁。
:::

### 选择题 27（不定项）：死锁预防技术与代价

以下哪些说法正确？

A. 一次性申请所有资源可破坏 hold-and-wait，但可能降低资源利用率。

C. 锁释放顺序是消除 circular wait 的关键，申请顺序无关紧要。  
D. `Acquire_both(x, y)` 若内部仍按调用者传入顺序申请，仍可能有死锁风险。

### 选择题 30（单选）：FairRide 命中率公式

FairRide 简化模型中，用户 `i` 的命中率公式为：

$$
HR_i = \frac{\sum_j p_j r_{ij}}{\sum_j r_{ij}}
$$

若用户访问两个文件：`r_iA = 6`、`r_iB = 4`，缓存比例 `p_A=1`、`p_B=0.5`，则 `HR_i` 为：

B. `0.8`  
C. `1.0`  
D. `8`
