你是一个资深前端设计与开发助手。请为一门课程搭建一个可部署在 GitHub Pages 的“课程笔记网站”。

====================
一、项目目标
====================
- 目标：构建一个简洁大方、风格统一、便于长期维护的课程笔记子站。
- 站点应支持：
  1) 首页展示课程信息与笔记列表
  2) Markdown 笔记美化渲染
  3) 中英文双语笔记与一键切换
  4) LaTeX 公式渲染
  5) 折叠备注（含多种类型样式）
- 该模板需要可迁移到其他课程，只需替换课程名、配色和笔记内容即可复用。

====================
二、目录与文件结构（必做）
====================
在仓库中创建一个课程目录（例如 `os/`，迁移时可换成 `db/`、`ml/`、`net/` 等），并至少包含：

1. `index.html`
- 课程首页
- 展示课程标题、课程描述、笔记列表
- 提供“如何添加笔记”入口（链接到 `note.html?file=addnote.md`）

2. `note.html`
- 笔记详情页
- 负责加载并渲染 Markdown
- 支持 URL 参数：
  - `?note=<NoteID>&lang=zh|en`（主模式）
  - `?file=<filename.md>`（兼容模式）

3. `render.js`
- 自定义 Markdown 解析器
- 支持标题、段落、列表、代码块、引用、分隔线、链接、行内代码、粗体/斜体
- 支持备注折叠语法（见“功能要求-备注”）

4. `style.css`
- 全站统一样式
- 首页与详情页风格一致
- 适配桌面与移动端

5. `addnote.md`
- 用户手册
- 说明如何新增笔记、挂载到首页、双语命名规则、公式与备注语法

6. 示例笔记（至少一讲）
- `Lec1.zh.md`
- `Lec1.en.md`

====================
三、功能要求（必须全部支持）
====================
A. 首页（index.html）
- 首页要有课程品牌头图区域（Hero）与笔记列表。
- 笔记列表由一个 NOTES 配置数组驱动，每条至少包含：
  - `id`
  - `titleZh`
  - `titleEn`
  - `summaryZh`
  - `updated`
- 点击列表项默认进入中文笔记：
  - `note.html?note=<id>&lang=zh`

B. 双语笔记系统
- 每个笔记必须有两份 Markdown：
  - `<NoteID>.zh.md`
  - `<NoteID>.en.md`
- 详情页右上角提供“中文 / English”切换按钮。
- 切换后仅变更 `lang` 参数，不破坏当前笔记 ID。

C. 一级标题独立渲染
- Markdown 的第一行一级标题（`# ...`）需从正文中抽离。
- 在页面中作为独立大标题渲染（正文不重复显示该标题）。

D. LaTeX 公式渲染
- 支持行内公式：`$...$`
- 支持块公式：`$$...$$`
- 推荐用 MathJax（CDN）进行渲染。
- 需在示例笔记中提供至少一个行内公式与一个块公式示例。

E. 备注折叠框（默认折叠）
- 支持语法：

  :::remark 标题
  内容...
  :::

- 点击标题可展开/折叠。
- 支持类型与别名（至少以下四类）：
  - `remark`（别名：`note`、`备注`）
  - `tip`（别名：`hint`、`提示`）
  - `warn`（别名：`warning`、`注意`）
  - `error`（别名：`danger`、`错误`）
- 不同类型需使用不同但和谐的颜色主题。
- 可在标题中使用少量 emoji（例如 📝 💡 ⚠️ ⛔）提升识别度。

F. 兼容性与错误处理
- 对 `file://` 场景给出友好提示（解释 fetch 受限）。
- 提示使用本地 HTTP 预览：
  - `python -m http.server 8000`
- 明确提示访问地址：
  - `http://localhost:8000/<course>/index.html`
- 对非法文件名或非法 note id 做安全校验。

====================
四、艺术设计与风格要求（重点）
====================
总体方向：
- 符合“课程学科定位”的视觉语言。
- 风格简洁大方、专业耐看，不花哨、不堆砌特效。

针对“操作系统类”课程建议：
- 深色科技基底（低饱和蓝绿系）
- 细网格 + 柔和光晕作为静态背景
- 终端/系统感元素（等宽字体用于编号、标签）

约束：
- 背景保持静态（不要动态粒子、不要快速动画）。
- 页面层级清晰，正文可读性优先。
- 配色应克制统一，不要刺眼高对比。
- 保持首页与详情页风格一致。

细节建议：
- 课程标题可做主副标题分行排版。
- 按钮、卡片、折叠框要有轻量 hover 反馈。
- 备注框颜色区分明确但不跳脱主色调。

====================
五、可维护性要求
====================
- 首页笔记数据集中在 NOTES 数组，新增笔记无需改多处。
- 语法规则写入 `addnote.md`，方便后续协作者接手。
- 样式与逻辑分离（CSS/JS 不混乱耦合）。
- 保证移动端可用（响应式布局）。

====================
六、验收清单（交付前逐条检查）
====================
1) 首页是否展示课程信息与笔记列表。
2) 点击 Lec1 是否能正常进入详情页。
3) 中英文切换是否可用（同一 NoteID 下切换 lang）。
4) 一级标题是否独立渲染且正文不重复。
5) LaTeX 行内与块公式是否成功渲染。
6) `remark/tip/warn/error` 折叠框是否默认折叠并可展开。
7) 各备注类型颜色是否区分且风格和谐。
8) `addnote.md` 是否完整说明新增流程。
9) `file://` 情况是否有友好提示与本地服务说明。
10) 移动端显示是否正常。

====================
七、迁移到其他课程时只需替换
====================
- 课程名、学期、课程描述
- 目录名（如 `os` -> `xxx`）
- NOTES 列表与笔记内容
- 色彩细节（保持整体简洁专业）

请严格按以上要求输出可直接运行的网页文件与示例内容。
