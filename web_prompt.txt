你是一个资深前端设计与开发助手。请为一门课程搭建一个可部署在 GitHub Pages 的“课程笔记网站”。

====================
一、项目目标
====================
- 目标：构建一个简洁大方、风格统一、便于长期维护的课程笔记子站。
- 站点应支持：
  1) 首页展示课程信息与笔记列表
  2) Markdown 笔记美化渲染
  3) 中英文双语笔记与一键切换
  4) LaTeX 公式渲染
  5) 折叠备注（含多种类型样式）
- 该模板需要可迁移到其他课程，只需替换课程名、配色和笔记内容即可复用。

====================
二、目录与文件结构（必做）
====================
在仓库中创建一个课程目录（例如 `os/`，迁移时可换成 `db/`、`ml/`、`net/` 等），并至少包含：

1. `index.html`
- 课程首页
- 展示课程标题、课程描述、笔记列表
- 可选提供“如何添加笔记”入口（`note.html?file=addnote.md`）

2. `note.html`
- 笔记详情页
- 负责加载并渲染 Markdown
- 支持 URL 参数：
  - `?note=<NoteID>&lang=zh|en`（主模式）
  - `?file=<filename.md>`（兼容模式）

3. `render.js`
- 自定义 Markdown 解析器
- 支持标题、段落、列表、代码块、引用、分隔线、链接、行内代码、粗体/斜体
- 支持备注折叠语法（见“功能要求-备注”）

4. `style.css`
- 全站统一样式
- 首页与详情页风格一致
- 适配桌面与移动端

5. `addnote.md`
- 用户手册
- 说明如何新增笔记、挂载到首页、双语命名规则、公式与备注语法

6. 示例笔记（至少一讲）
- `Lec1.zh.md`
- `Lec1.en.md`

====================
三、功能要求（必须全部支持）
====================
A. 首页（index.html）
- 首页要有课程品牌头图区域（Hero）与笔记列表。
- 笔记列表由一个 NOTES 配置数组驱动，每条至少包含：
  - `id`
  - `titleZh`
  - `titleEn`
  - `summaryZh`
  - `updated`
- 点击列表项默认进入中文笔记：
  - `note.html?note=<id>&lang=zh`

B. 双语笔记系统
- 每个笔记必须有两份 Markdown：
  - `<NoteID>.zh.md`
  - `<NoteID>.en.md`
- 详情页右上角提供“中文 / English”切换按钮。
- 切换后仅变更 `lang` 参数，不破坏当前笔记 ID。

C. 一级标题独立渲染
- Markdown 的第一行一级标题（`# ...`）需从正文中抽离。
- 在页面中作为独立大标题渲染（正文不重复显示该标题）。

D. LaTeX 公式渲染
- 支持行内公式：`$...$`
- 支持块公式：`$$...$$`
- 推荐用 MathJax（CDN）进行渲染。
- 需在示例笔记中提供至少一个行内公式与一个块公式示例。

E. 备注折叠框（默认折叠）
- 支持语法：

  :::remark 标题
  内容...
  :::

- 点击标题可展开/折叠。
- 支持类型与别名（至少以下四类）：
  - `remark`（别名：`note`、`备注`）
  - `tip`（别名：`hint`、`提示`）
  - `warn`（别名：`warning`、`注意`）
  - `error`（别名：`danger`、`错误`）
- 不同类型需使用不同但和谐的颜色主题。
- 可在标题中使用少量 emoji（例如 📝 💡 ⚠️ ⛔）提升识别度。

F. 兼容性与错误处理
- 对 `file://` 场景给出友好提示（解释 fetch 受限）。
- 提示使用本地 HTTP 预览：
  - `python -m http.server 8000`
- 明确提示访问地址：
  - `http://localhost:8000/<course>/index.html`
- 对非法文件名或非法 note id 做安全校验。

====================
四、艺术设计与风格要求（重点）
====================
总体方向：
- 必须符合课程学科定位，视觉语言要“像这门课”，而不是同一模板换标题。
- 风格可简洁，但不能平庸；需要有清晰的主创意（hero concept）。
- 允许动态背景，但必须低干扰、低频率、服务内容阅读。

核心设计原则（必须执行）：
- 先列出该课程相关的视觉元素候选（至少 4-6 个）。
- 从候选中只选 **1 个核心元素** 作为主视觉（例如“证明树”或“等高线图”）。
- 主视觉需要贯穿首页与详情页，形成统一识别。
- 不要同时堆多个同类元素（例如“关键词漂浮 + 关键词雨 + 卡片漂浮”）。
- 明确避免模板化迁移：不要只把上一门课的动效换文案继续用。

风格方差要求（跨课程）：
- 不同课程的版式、配色、氛围应明显不同，避免“换皮同款”。
- 不要默认深色主题；可使用浅色、纸面、实验室白板、仪表盘等风格。
- 不要默认网格背景；网格只在学科语义合理时使用（如 OS）。
- 字体策略可随课程变化：例如逻辑课可偏讲义/衬线，优化课可偏图表/工程感。

创作流程（建议严格按顺序）：
1) 先写“元素候选清单”（与课程语义强关联）。
2) 选 1 个最具辨识度且实现成本可控的核心元素。
3) 设计页面构图（不是只改颜色）：例如讲义双栏、仪表盘卡片、论文页边注等。
4) 再实现动态层（如有），确保不遮挡正文且支持 `prefers-reduced-motion`。
5) 最后统一组件语言（按钮、卡片、折叠框、标题层级）。

避免事项（强约束）：
- 不要重复“关键词漂浮/代码雨/卡片漂浮”这种通用套路，除非与课程高度匹配且只保留一个主视觉。
- 不要让背景动画喧宾夺主，不要高速闪动。
- 不要让所有课程都长得像“操作系统站点”。

示例（仅作方向，不是固定模板）：
- 操作系统：终端窗口/调度感元素（可深色科技风）。
- 逻辑导论：证明树或推理结构（可讲义/纸面风）。
- 凸分析与优化：等高线/下降轨迹（可图表仪表盘风）。

细节建议：
- 课程标题可做主副标题分行排版。
- 按钮、卡片、折叠框保持轻量交互反馈。
- 备注框颜色分层明确，但需与课程主配色和谐。
- 背景与内容层级要明确，正文可读性优先。

====================
五、可维护性要求
====================
- 首页笔记数据集中在 NOTES 数组，新增笔记无需改多处。
- 语法规则写入 `addnote.md`，方便后续协作者接手。
- 样式与逻辑分离（CSS/JS 不混乱耦合）。
- 保证移动端可用（响应式布局）。

====================
六、验收清单（交付前逐条检查）
====================
1) 首页是否展示课程信息与笔记列表。
2) 点击 Lec1 是否能正常进入详情页。
3) 中英文切换是否可用（同一 NoteID 下切换 lang）。
4) 一级标题是否独立渲染且正文不重复。
5) LaTeX 行内与块公式是否成功渲染。
6) `remark/tip/warn/error` 折叠框是否默认折叠并可展开。
7) 各备注类型颜色是否区分且风格和谐。
8) `addnote.md` 是否完整说明新增流程。
9) `file://` 情况是否有友好提示与本地服务说明。
10) 移动端显示是否正常。
11) 是否先列元素候选并仅选择 1 个核心视觉。
12) 不同课程之间是否有明显风格差异，而非同模板换色。

====================
七、迁移到其他课程时只需替换
====================
- 课程名、学期、课程描述
- 目录名（如 `os` -> `xxx`）
- NOTES 列表与笔记内容
- 核心视觉元素（每门课重新选择，不复用上一门课）
- 配色与版式（保证跨课程风格方差）

请严格按以上要求输出可直接运行的网页文件与示例内容。
